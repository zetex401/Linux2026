# HW03 — Answers

## 1. Что хранит inode?
Inode хранит метаданные файла: размер, права доступа, владельца, timestamps и указатели на блоки данных. Он НЕ хранит имя файла. Имя хранится в директории как запись «имя → inode». :contentReference[oaicite:0]{index=0}

## 2. ln file.txt link.txt → rm file.txt
Данные НЕ потеряны. Потому что создаётся hard link — второе имя на тот же inode. При удалении file.txt уменьшается link count, но данные остаются, пока есть хотя бы одна ссылка.

## 3. Почему df ≠ du
1) Удалённые, но открытые файлы (df видит, du нет)  
2) Зарезервированные блоки ext4  
3) Sparse files — логический размер больше физического

## 4. ordered vs writeback
В режиме ordered сначала записываются данные, затем метаданные — это безопаснее. В writeback порядок не гарантируется — быстрее, но риск потери данных выше. По умолчанию используется ordered.
