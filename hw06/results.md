 Ученная гипотеза псевдоумника

Я ожидаю, что процесс с nice -10 завершится быстрее, чем процесс с nice +10. Это связано с тем, что более низкое значение nice означает более высокий приоритет для CPU.

Анализ идет у нас ниже
Планировщик Linux CFS распределяет CPU между процессами с учетом их nice-значений. Процесс с nice -10 имеет больший вес и должен получать больше процессорного времени. Процесс с nice +10 имеет меньший приоритет, поэтому при фоновой нагрузке он может завершиться позже. Разница особенно заметна, когда CPU занят несколькими процессами одновременно.

Вопрос что изменится без фоновой нагрузки? 
Без фоновой нагрузки разница между процессами может стать меньше. Если CPU свободен, оба процесса могут получить достаточно процессорного времени. Поэтому nice сильнее влияет на результат именно тогда, когда есть конкуренция за CPU.


Результаты
nice -10:

real    0m0.437s
user    0m0.000s
sys     0m0.401s

nice +10:

real    0m0.431s
user    0m0.001s
sys     0m0.397s
