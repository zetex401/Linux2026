Вы создаёте AI-агент: Python-программу, которая принимает запрос на естественном языке, вызывает инструменты (tools) для работы с файловой системой и системными данными, и возвращает осмысленный ответ.

7. Telegram-интеграция

Логика такая:

Минимальный запуск Telegram-бота:

from telegram.ext import Application, CommandHandler, MessageHandler, filters

async def start(update, context):
    await update.message.reply_text("Linux FS AI Agent запущен. Напишите команду.")

async def handle_message(update, context):
    user_text = update.message.text
    answer = run_agent(user_text)
    await update.message.reply_text(answer)

app = Application.builder().token(TELEGRAM_BOT_TOKEN).build()
app.add_handler(CommandHandler("start", start))
app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, handle_message))
app.run_polling()

9. Retry при 429/503

На защите могут спросить:

Что происходит, если Gemini API временно недоступен?

Ответ:

В коде реализован retry. Если API возвращает 429 или 503, агент ждёт несколько секунд и повторяет запрос. Это защищает от временных ошибок лимита или перегрузки сервиса.

Пример логики:

import time

for attempt in range(3):
    try:
        response = client.models.generate_content(...)
        return response
    except Exception as e:
        if "429" in str(e) or "503" in str(e):
            time.sleep(2)
            continue
        raise e

Проблема с погодой в боте

Если бот неправильно показывает погоду, причины обычно такие:

1. Неправильный город

Например пользователь пишет:

Алматы

А API ищет:

Almaty

Лучше внутри бота сделать словарь:

Алматы → Almaty
Астана → Astana
Шымкент → Shymkent
Токио → Tokyo
Лондон → London
2. API вернул другой город

Некоторые weather API могут найти не тот город, особенно если город есть в разных странах.

Например:

Moscow
Paris
London

могут иметь несколько совпадений.

Лучше указывать страну:

Almaty,KZ
Astana,KZ
Shymkent,KZ
3. Неправильные единицы температуры

API может отдавать температуру в Kelvin.

Например:

295.15 K

Это примерно:

22°C

Для Celsius нужно добавить параметр:

units=metric
4. Неправильный язык

Чтобы описание погоды было на русском:

lang=ru
5. Неправильный API key

Если ключ неправильный или не экспортирован:

echo $WEATHER_API_KEY

Если пусто — бот не видит ключ.

Нужно:

export WEATHER_API_KEY="твой_weather_key"
6. Бот берёт старые данные

Если в коде есть кеширование, бот может показывать старую погоду.

Решение: каждый запрос должен заново обращаться к API.

7. Ошибка timezone

Погода может быть правильной, но время рассвета/заката неправильное, если бот не учитывает timezone города.

Для времени лучше использовать отдельный timezone tool, например:

Asia/Almaty
Asia/Tokyo
Europe/London
America/New_York
Правильный weather-запрос

Пример для OpenWeather:

curl "https://api.openweathermap.org/data/2.5/weather?q=Almaty,KZ&appid=$WEATHER_API_KEY&units=metric&lang=ru"

Через jq:

curl -s "https://api.openweathermap.org/data/2.5/weather?q=Almaty,KZ&appid=$WEATHER_API_KEY&units=metric&lang=ru" | jq

Достать температуру:

curl -s "https://api.openweathermap.org/data/2.5/weather?q=Almaty,KZ&appid=$WEATHER_API_KEY&units=metric&lang=ru" | jq '.main.temp'

1. Системный сбор данных на C

Вместо модулей os в Python, ты пишешь небольшие утилиты на C, которые читают ядро напрямую.
Создай файл sys_monitor.c:

C


#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
    char buffer[256];
    // Системный вызов open для чтения из виртуальной ФС
    int fd = open("/proc/loadavg", O_RDONLY);
    if (fd < 0) {
        perror("Error opening /proc/loadavg");
        return 1;
    }
    int bytes_read = read(fd, buffer, sizeof(buffer) - 1);
    buffer[bytes_read] = '\0';
    printf("Load Average: %s", buffer);
    close(fd);
    return 0;
}

Скомпилируй его: gcc sys_monitor.c -o sys_monitor

#!/bin/bash

# 1. Собираем системный контекст
SYS_LOAD=$(./sys_monitor)
DISK_USAGE=$(df -h / | tail -n 1)
USER_QUERY="Почему у меня тормозит система?"

# 2. Безопасно формируем JSON-запрос с помощью jq (чтобы избежать проблем с кавычками)
JSON_PAYLOAD=$(jq -n \
  --arg prompt "$USER_QUERY" \
  --arg ctx "Контекст: $SYS_LOAD | Диск: $DISK_USAGE" \
  '{
    "contents": [{
      "parts": [{"text": ($prompt + "\n" + $ctx)}]
    }]
  }')

# 3. Отправляем запрос через curl
API_KEY="ТВОЙ_КЛЮЧ"
URL="https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key=$API_KEY"

RESPONSE=$(curl -s -X POST "$URL" -H 'Content-Type: application/json' -d "$JSON_PAYLOAD")

# 4. Извлекаем ответ (текст) из огромного JSON ответа

if [[ "$choice" == "Y" || "$choice" == "y" ]]; then

fi
Как перенести твой Tool Calling?

В Python ты передавал tools=[tool_config]. В curl ты тоже можешь передать массив tools в JSON, но парсить вызов функции (Function Call) в Bash придётся руками:
1. Ты проверяешь через jq, есть ли в ответе поле functionCall:
TOOL_NAME=$(echo "$RESPONSE" | jq -r '.candidates[0].content.parts[0].functionCall.name')
2. Если TOOL_NAME не null, ты извлекаешь аргументы (тоже через jq).
3. Запускаешь нужную bash-команду или C-утилиту через case "$TOOL_NAME" in ... esac.
